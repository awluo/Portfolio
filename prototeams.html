PROTOTEAMS

A webapp to prototype teams

Overview

Forming teams is often challenging. There are many factors to consider when selecting team members: personality, skills, etc. But which ones really matter? As a tool to facilitate face-to-face, small group interactions, ProtoTeams eases the process of finding teams by helping you meet with potential members and "prototype" teams before choosing members you want to work with.

Duration
Summer 2019 (3 months)

Team
Gus Umbelino, Sam Blake, Amy Luo

Skills
UX Design, User Testing, Research
PROBLEM
How can we facilitate and ease the process of team formation?
SOLUTION
ProtoTeams is a system designed to help people meet potential teammates through a series of short, small group activities. Based on the concept of "speed dating", this app allows people to interact with various temporary groups, before committing to a long-term team.

How It Works

ProtoTeams is divided into 3 components: a participant mobile interface, a shared display, and a host dashboard. The shared display is displayed or projected on a large screen and is controlled by the host. Each session has a certain number of rounds, each containing a number of activities, Sessions can be customized (group size, number of rounds, types of activities) by the host through the dashboard.

Shared Host Display
1.

Join the session by entering the corresponding unique code generated on the shared display.

2.

Form groups by finding others with the same icon on their screen

3.

Participate in group activities and discussion questions. Timer in the top corner as well as Team member drop down menu.

4.

After each round, assess how well you got to know each member. If there are more rounds left, repeat group formation with a new set of people.

Process

User Testing
We tested the new implementation in an undergraduate project-based design course at UCSD where students needed to find teammates for a quarter-long project. We conducted three pilot studies with the new tool during the first few weeks of class. Quantitative data (timing, member ratings) was collected through the app, and qualitative data (feedback, impressions) was collected through a post-session survey as well as through user interviews
Students looking for their groups
Feedback

Response
Following the user testing sessions and feedback, we updated the app accordingly with the following design changes:

  • Increased time for group activities
  • Updated shape icons to appear more vibrant and with distinctive colors
  • Added transition screen when it is time to leave group

Team Confirmation

An issue that arose was trying to find a method for confirming group members had physically located each other. Previously, individuals could simply press a button to confirm their team was found, regardless if this was true or not. We considered various methods and eventually devided to assign each member an individual number of oranges and require the group sum to be entered as a means to ensure everyone was located.
